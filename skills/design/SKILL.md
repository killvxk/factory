# /design — Build It With a Point of View

## Goal

Produce UI that has a committed aesthetic direction, not generic defaults.
If it could have been generated by a default prompt, it is not good enough.

## Constraints

- Lock the direction FIRST via 4 mandatory questions, before any code
- Inter, Roboto, and system-ui are BANNED as primary display fonts ("They are invisible")
- Must open in a browser and visually verify — never claim "it looks right" without seeing it
- Default to cardless sections; require atmosphere on backgrounds (mesh, noise, texture)

## Direction Lock (4 Required Questions)

Before writing any UI code, ask:
1. **Who** uses this and in what context?
2. **Aesthetic direction** — named precisely. "Clean and modern" is rejected.
   Examples: "dense editorial", "raw terminal", "brutalist grid", "warm artisanal"
3. **The one thing** it leaves in memory after closing the tab
4. **Hard constraints** (framework, accessibility, existing design system, responsive breakpoints)

After answers received, proceed. Do NOT re-ask.

## Anti-Template Test

After building, apply this test:
If you swapped in completely different content and the layout still made sense unchanged,
you built a TEMPLATE, not a design. Redo it.

## Forbidden Defaults

These generic layouts must NOT appear unless explicitly requested:
- Purple/blue gradient hero section
- Three identical cards in a row
- Nav with logo-center-right pattern
- Alternating white/gray sections
- Centered icon + heading + paragraph blocks
- Four-column footer grid

## Motion Budget

- Entrance animations: 200-400ms
- Exit animations: 150-250ms
- Test: if removing the animation changes nothing about how the page feels, remove it

## Verification

- Open the result in a browser (use Bash to serve, or write an HTML file)
- Take a screenshot or describe what you see
- Confirm it matches the locked direction
- Check mobile viewport if responsive was a constraint

## Gotchas

- Used Inter as display font — it disappears into the background
- Generated a template (identical cards, identical shadows, identical padding)
- Claimed it looked right without opening a browser
- Glassmorphism on mobile without checking backdrop-filter performance cost
- Colors that don't commit ("muted blue-gray that goes with everything")
- Layout jittered on state transitions — all states must occupy same layout footprint
