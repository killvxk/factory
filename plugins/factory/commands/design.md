# /factory:design

## Preflight

- Verify the task affects user-facing UI or visual presentation.
- Verify the target surface and constraints.

## Plan

- Pick a clear aesthetic direction before editing implementation details.
- Identify typography, color, layout, motion, and atmospheric choices.

## Commands

- Audit the current UI and describe what should change.
- Define one committed direction instead of averaging several safe options.

## Verification

- The proposed direction should feel distinctive, implementable, and aligned with the product goal.

## Summary

- Result: design direction chosen.
- Status: success only when the direction is clear enough to build.

## Next Steps

- Move to `/factory:build` for implementation.
